### แล้วไง ?

ในเมื่อ agent (ai agents) เขียนโค้ดได้ดีกว่า ก็ให้มันเขียนไปสิ คำกล่าวนี้อาจถูกใจบางคน ในขณะเดียวกันก็อาจจะขัดใจบางคน .. ส่วนตัวผมนั้นหรอ .. แอบกังวลนิดๆ ไม่ได้กังวลว่าจะตกงานนะ แต่กังวลว่าตัวผมเอง อาจจะขี้เกียจตรวจเช็คโค้ดที่ agent สร้าง แล้วโค้ดที่ผมสร้างอาจจะเป็นภาระของทั้งตัวผมเอง และคนอื่น ในอนาคต .. ช่วงสองสามปีมานี้ ผมเจอโค้ดเบสขนาดใหญ่ (ระดับหมื่นบรรทัด → สำหรับผม ผมว่าใหญ่) ที่แม่งโครตจะอ่านไม่รู้เรื่อง ส่วนนึงก็เป็นเพราะผมด้วย (ผมก็มีส่วนร่วมในการสร้าง legacy code ชุดนั้น)

<br>

ผมมองว่าท้ายที่สุดการ vibe coding ไปเรื่อยๆ (โดยที่ขาดสติ) แม้เราอาจมองว่ามันเร็วและมีคุณภาพนั้น (เช่น มองว่ามันเขียนโค้ดดีกว่าเราไรงี้) แต่ส่วนตัวผมมองว่าเราอาจมีหนีทางความรู้โดยไม่รู้ตัว .. มันเร็ว มันดีย์ .. แต่เราต้องรู้ทันมัน (อย่าคิดว่าเราเก่ง agent ต่างหากที่เก่ง ไม่ใช่เรา 😱) จาก ปสก ส่วนตัวที่ผมเจอมาก็คือพอกลับมาอ่านโค้ดก็มักจะ งงๆ ว่านี่เรา (หรือ agent) ทำอะไรลงไป 😵‍💫 .. เหมือนความเข้าใจในโค้ดมีอายุสั้นลง **จุดนี้สะท้อนให้ตัวผมเองตระหนักถึงการใช้ agent อย่างมีสติ และหันมาอัดพื้นฐานให้แน่นอยู่เสมอ แม้ว่าผมจะขี้เกียจแค่ไหนก็ตาม ฮีบบบ .. แม้ว่าจะมีเครื่องทุ่นแรงอย่าง agent ก็ตาม ก็ขอให้มันทุ่นแค่แรงกาย (มันช่วยลดอาการเมื่อยนิ้วได้เยอะจริงๆ) ก็พอ อย่าทุ่นแรงสมองเลย**

<br>

แล้วอนาคตของสายเดฟหละ (จริงๆก็ปัจจุบันละนะ 😅) เราก็คงใช้ agent แทบจะ 100% นั่นแหละ แต่คงต้องใช้อย่างมีสติมากขึ้น .. และนอกจากเราจะต้องบูรณาการมันเข้ากับทั้งกระบวนการพัฒนาซอฟแวร์ เราอาจจะต้องบูรณาการมันเข้ากับกระบวนการคิดของเราด้วย (ผมมองว่าตรงนี้ แต่ละคนอาจแตกต่างกัน .. ก็ไปจูน agent ของตัวเองเอา)
